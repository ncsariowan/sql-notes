SQL Keywords

Select

SELECT column1, column2… FROM table1
SELECT * FROM table1

Select Distinct

Only shows unique values

SELECT DISTINCT column1, column2… FROM table1

SELECT COUNT(DISTINCT column1) FROM table1

Where

Qualify selects with a conditional.

SELECT column1 FROM table1 WHERE conditional

Ex:
SELECT * FROM table1 WHERE column1='something'
SELECT * FROM table1 WHERE column1 BETWEEN 0 AND 60
SELECT * FROM table1 WHERE column1 IN ('One Value', 'Another Value')
SELECT * FROM table1 WHERE column1 LIKE '%s' (patterns)

You can use =, <, >, >=, <=, != too.

And, Or and Not

SELECT * FROM table1 WHERE conditional1 AND conditional2 AND ...
SELECT * FROM table1 WHERE conditional1 OR conditional2 OR...
SELECT * FROM table1 WHERE NOT conditional

Order By

SELECT column1, column2 ... FROM table1 ORDER BY column1, column2 ... ASC|DESC

Insert Into

INSERT INTO table1 ( `column1`, `column2`, ...) VALUES ("value1", "value2", ...)

Is Null and Is Not Null

SELECT column1, column2... FROM table1 WHERE column1 IS NULL
SELECT column1, column2... FROM table1 WHERE column1 IS NOT NULL

Update

UPDATE table1 SET column1 = value1, column2 = value2, ... WHERE conditional

Delete

DELETE FROM table1 WHERE conditional1

Top and Limit

Select a specified number of entries

SQL Server / MS Access Syntax:
SELECT TOP number|percent column1, column 2 ... FROM table1 WHERE conditional
MySQL Syntax
SELECT column1, column 2.. FROM table1 WHERE condition LIMIT number

SELECT TOP number PERCENT column1, column2 ... FROM table1

Protip: add a WHERE clause to the end of these!

Min and Max

SELECT MIN(column1) FROM table1 (WHERE conditional)
SELECT MAX(column1) FROM table1 (WHERE conditional)

Count, Average and Sum

SELECT COUNT(column1) FROM table1 (WHERE conditional)
SELECT AVG(column1) FROM table1 (WHERE conditional)
SELECT SUM(column1) FROM table1 (WHERE conditional)

Like

SELECT column1, column2 ... FROM table1 WHERE columnN LIKE pattern

% - zero, one, or multiple chars
_ - single char

LIKE Operator Description
'a%' starts with ‘a’
'%a' ends with ‘a’
'%or%' has ‘or’ in any position
'_r%' has ‘r’ in second position
'a__%' starts with ‘a’, at least three chars in length
'a%o' starts with ‘a’, ends with ‘o’

In

SELECT column1, column2 ... FROM table1 WHERE columnN IN (value1, value2 ...)
SELECT column1, column2 ... FROM table1 WHERE columnN IN (SELECT statement)

Between

SELECT column1, column2 ... FROM table1 WHERE columnN BETWEEN value1 AND value2

As (Aliases)

SELECT column1 AS alias FROM table1

Joins

SELECT table1.columnA, table2.columnB, ... FROM table1 JOIN table2 ON table1.columnN=table2.columnN

Inner Joins

Only entries with matching values
SELECT table1.columnA, table2.columnB, ... FROM table1 INNER JOIN table2 ON table1.columnN=table2.columnN

Left Joins

Everything on left table plus matching values entries from right table joined
SELECT table1.columnA, table2.columnB, ... FROM table1 LEFT JOIN table2 ON table1.columnN=table2.columnN

Left Joins

Everything on right table plus matching values entries from left table joined
SELECT table1.columnA, table2.columnB, ... FROM table1 RIGHT JOIN table2 ON table1.columnN=table2.columnN

Full Joins

Everything is joined
SELECT table1.columnA, table2.columnB, ... FROM table1 RIGHT JOIN table2 ON table1.columnN=table2.columnN

Self Joins

SELECT table1.columnA, table2.columnB, ... FROM table1 T1, table2 T2 WHERE condition

Databases Notes

Relational Database Management System (RDBMS)

Record = Entry = Row
Unique Key: Each value must be unique
Primary Key: Key that doesn’t have null values
Foreign Key: Entry in one table that serves as a primary key in another

Normalizing Data

First Normal Form: Each entry should only have one piece of data in it
Second Normal Form: Each field in a row should be fully dependent on a single key
Third Normal Form: All data in non-key columns should be independent of each other
